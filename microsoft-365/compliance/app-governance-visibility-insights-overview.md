# <a name="learn-about-visibility-and-insights"></a>Informationen über Transparenz und Insights

>*[Microsoft 365-Lizenzierungsleitfaden für Sicherheit und Compliance](https://aka.ms/ComplianceSD).*

Mit der Microsoft App-Governance können Sie schnell Transparenz und aussagekräftige Insights in Ihr Microsoft 365-Anwendungsökosystem gewinnen. Sie starten mit dem App-Governance-Dashboard, das eine Übersicht über die Benachrichtigungen und Apps in Ihrem Mandanten bietet, welche die Aufmerksamkeit des Administrators erfordern.

Über die App-Governance-Sichtbarkeit und -Insights können Sie Folgendes sehen:

- Eine Liste der OAuth-fähigen Apps, die auf Microsoft 365-Daten via Microsoft Graph zugreifen.
- Eine umfassende Sicht auf App-Aktivitäten, so dass Sie darauf reagieren oder antworten können.

>[!Note]
>Nur-Azure-Apps, denen keine Berechtigungen für den Zugriff auf Microsoft 365-Ressourcen gewährt wurden, werden in der App-Governance nicht angezeigt.
>

Lesen Sie [Administratorrollen](app-governance-get-started.md#administrator-roles) für einen Überblick der benötigten Administratorrollen für Sichtbarkeit und Insights.

Mit der App-Governance können Sie Folgendes sehen:

- Ein Dashboard mit allen Insights.
- Daten, auf die von einzelnen und allen Apps zugegriffen wird, mit Insights in die Auslastung und auf Benutzerebene.
- App-Informationen und -Metadaten, beispielsweise Berechtigungen, Registrierungsdatum und Zertifizierung.
- Herausgeberinformationen und -Metadaten, beispielsweise der Name und der Verifizierungsstatus.
- Verwendung der Top-Ressourcen (E-Mails und Dateien) im gesamten Mandanten.
- Insights über:

  - Hochprivilegierte Apps.
  - Überprivilegierte Apps.
  - App mit hoher Nutzung.
  - Benutzer mit der höchsten Zustimmung, auf dessen Daten eine bestimmte App zugreifen kann.
  - Prioritätskonten, die Daten enthalten, auf welche eine bestimmte App zugreifen kann.

- Eine kumulative Ansicht der Benutzer, die auf Apps zugreifen.
- Benachrichtigungs-Insights.
- Richtlinienlisten-Insights.
<!-->
- Richtlinien, die in MCAS im App-Governance-Portal erstellt wurden.
-->
- Benachrichtigungen für OAuth-Apps, die in MCAS erzeugt wurden, im App-Governance-Portal.

Sie können auch Folgendes tun:

- Aufschlüsseln auf eine einzelne App (App-Seite) mit allen zugehörigen Einblicken.
- Aufschlüsseln nach Top-Benutzern nach Daten, und Prioritätskonten innerhalb einer einzelnen App.
