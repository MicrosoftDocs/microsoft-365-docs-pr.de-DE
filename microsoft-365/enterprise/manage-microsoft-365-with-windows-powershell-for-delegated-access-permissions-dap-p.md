# <a name="how-to-manage-microsoft-365-with-windows-powershell-for-delegated-access-permissions-partners"></a>Verwalten von Microsoft 365 mit Windows PowerShell für Partner mit delegierten Zugriffsberechtigungen

*Dieser Artikel gilt sowohl für Microsoft 365 Enterprise als auch für Office 365 Enterprise.*

DAP-Partner (Delegated Access Permission, delegierte Zugriffsberechtigung) sind Syndication-Partner und Cloudlösungsanbieter (Cloud Solution Providers, CSP). Viele sind Netzwerk- oder Telekommunikationsanbieter. Sie bündeln Microsoft 365 Abonnements in ihren Serviceangeboten. Wenn sie ein Microsoft 365-Abonnement verkaufen, erhalten sie automatisch Die Berechtigung Verwalten im Auftrag von (AOBO) für die Kundenmandschaften, damit sie diese Vermandungen verwalten und melden können. Diese Aufgaben sind im Admin Center Microsoft 365 schwierig. Es ist viel einfacher, PowerShell für Microsoft 365 verwaltungstechnische Aufgaben auszuführen, z. B.:
- Auflisten aller **Kunden-TenantIds und** ihrer Domänen 
- Identifizieren aller Benutzer in einem Kunden-Mandanz und den zugewiesenen Lizenzen
> [!NOTE]
> Einige administrative Aufgaben können nur in PowerShell ausgeführt werden.
