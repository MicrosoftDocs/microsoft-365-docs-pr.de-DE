|Feature|Beschreibung|
|---|---|
|**Schutz vor Schadsoftware**||
|Schadsoftwareschutz durch Einsatz mehrerer Module|Durch mehrere Antischadsoftwaremodule sind unsere Kunden jederzeit automatisch geschützt.|
|Immer-on-Schadsoftwarefilterung|Sie können die Schadsoftwarefilterung nicht deaktivieren. Wir sind der Ansicht, dass ein konsistentes und strenges Sicherheitsniveau für alle Kunden ein wichtiger Bestandteil der notwendigen tiefgreifenden Verteidigung zum Schutz Ihrer E-Mail-Umgebung ist. Infolgedessen ist die Schadsoftwarefilterung für alle Kunden automatisch aktiviert.|
|Prüfung des Nachrichtentexts und der Anlagen auf Schadsoftware|Der Dienst untersucht die aktive Nutzlast im Nachrichtentext und in allen Anlagen auf Schadsoftware.|
|Antispywareschutz|Antischadsoftwareschutz umfasst Virenschutz und Antispywareschutz.|
|Richtlinien für Schadsoftwarefilter|Jede Organisation verfügt über eine Standardmäßige Antispamrichtlinie, die für alle Empfänger gilt. Für eine höhere Granularität können Sie benutzerdefinierte An malware-Richtlinien erstellen, die für bestimmte Benutzer, Gruppen oder Domänen in Ihrer Organisation gelten. Benutzerdefinierte Richtlinien werden immer vor der Standardrichtlinie angewendet, Sie können jedoch die Reihenfolge ändern, in der benutzerdefinierte Richtlinien angewendet werden. <p> Sie können die folgenden Einstellungen in Anti-Malware-Richtlinien konfigurieren: <ul><li>**Allgemeine Anlagenfilterung:** Aktivieren Sie eine angepasste Liste von Dateitypen, die immer als Schadsoftware angenommen werden.</li><li>**ZAP für Schadsoftware:** Rückwirkend in Quarantäne übermittelte Schadsoftwarenachrichten. Weitere Informationen finden Sie unter [Zero-Hour Auto Purge (ZAP) in Exchange Online](zero-hour-auto-purge.md).</li><li>**Empfängerbenachrichtigungen:** Isolieren Sie die Nachricht automatisch, oder isolieren Sie die Nachricht, und senden Sie sie auch, wenn alle Anlagen durch eine einzelne Textdatei ersetzt werden, die Standard- oder benutzerdefinierten Text enthält.</li><li>**Absenderbenachrichtigungen:** Benachrichtigen Sie Absender, dass ihre Nachricht als Schadsoftware erkannt wurde.</li><li>**Administratorbenachrichtigungen:** Benachrichtigen Sie einen Administrator, wenn Nachrichten von internen oder externen Absendern als Schadsoftware erkannt wurden.</li></ul> <p> Weitere Informationen finden Sie unter [Configure anti-malware policies](configure-anti-malware-policies.md).|
|**Antiphishingschutz**||
|Antiphishingschutz|EOP verwendet eine Liste der Domänen, die von bekannten Spammern verwendet werden.|
|Antispoofingschutz|Antiphishingschutz in EOP umfasst Anti-Spoofing-Schutz. Weitere Informationen finden Sie unter [Anti-Spoofing Protection](anti-spoofing-protection.md). <p> Antiphishingschutz in Microsoft Defender for Office 365 umfasst auch Identitätswechselschutz. Weitere Informationen finden Sie unter [Exklusive Einstellungen in Antiphishingrichtlinien in Microsoft Defender für Office 365](set-up-anti-phishing-policies.md#exclusive-settings-in-anti-phishing-policies-in-microsoft-defender-for-office-365).|
|**Antispamschutz**||
|Eingehende Spamerkennung|Weitere Informationen finden Sie unter [Antispamschutz in Microsoft 365](anti-spam-protection.md). <p> Weitere Schritte, die in Hybridumgebungen erforderlich sind, finden Sie unter [Configure EOP to deliver spam to the Junk Email folder in hybrid environments](/exchange/standalone-eop/configure-eop-spam-protection-hybrid).|
|Schutz vor Rückscatter|Weitere Informationen finden Sie unter [Backscatter und EOP](backscatter-messages-and-eop.md).|
|Massen-E-Mail-Filterung|EOP verwendet den Schwellenwert für Massenbeschwerde (Bulk Complaint, BCL) in Antispamrichtlinien, um Massen-E-Mail-Nachrichten als Spam zu kennzeichnen. Weitere Informationen hierzu finden Sie in den folgenden Themen: <ul><li>[Worin besteht der Unterschied zwischen Junk-E-Mail und Massen-E-Mail?](what-s-the-difference-between-junk-email-and-bulk-email.md)</li><li>[Massenbeschwerdestufe (Bulk Complaint Level, BCL) in EOP](bulk-complaint-level-values.md)</li><li>[Konfigurieren von Antispamrichtlinien](configure-your-spam-filter-policies.md)</li></ul>|
|Sperrlisten für schädliche URLs|EOP verwendet mehrere URL-Sperrlisten, die Unterstützung beim Erkennen bekannter schädlicher Links in Nachrichten bieten.|
|**Ausgehender Spamschutz**||
|Ausgehende Spamerkennung|Ausgehender Antispamschutz ist immer aktiviert, wenn Sie den Dienst zum Senden ausgehender E-Mails verwenden. Weitere Informationen finden Sie unter [Ausgehender Spamschutz](outbound-spam-controls.md).|
|Richtlinien für ausgehende Spamnachrichten|Jede Organisation verfügt über eine standardmäßige ausgehende Spamrichtlinie, die für alle Empfänger gilt. Für eine höhere Granularität können Sie benutzerdefinierte Antispamrichtlinien erstellen, die für bestimmte Benutzer, Gruppen oder Domänen in Ihrer Organisation gelten. Benutzerdefinierte Richtlinien werden immer vor der Standardrichtlinie angewendet, Sie können jedoch die Reihenfolge ändern, in der benutzerdefinierte Richtlinien angewendet werden. <p> Sie können die folgenden Einstellungen in Antispamrichtlinien konfigurieren: <ul><li>**Messsage limts**: Sie können Grenzwerte [](/office365/servicedescriptions/exchange-online-service-description/exchange-online-limits#receiving-and-sending-limits) festlegen, die niedriger sind als die Diensteinstellungen für externe Empfänger pro **Stunde,** interne Empfänger **pro** Stunde und maximale Anzahl von Empfängern **pro Tag**</li><li>**Aktion für Benutzer, die** einen Grenzwert überschreiten: Beschränken Sie den Benutzer für 24 Stunden, beschränken Sie den Benutzer bis zur Veröffentlichung, oder warnen Sie nur.</li><li>**Aktivieren oder Deaktivieren der automatischen externen E-Mail-Weiterleitung**: [Weitere Informationen](external-email-forwarding.md)</li><li>**Benachrichtigen oder Senden von Kopien von Nachrichten an Administratoren**</li></ul> <p> Weitere Informationen finden Sie unter [Configure outbound spam filtering in EOP](configure-the-outbound-spam-policy.md).|
|**Verbindungsfilterung**||
|Verbindungsfilterrichtlinie|Konfigurieren Sie die Liste der zulässigen IP-Adressen und die Liste "IP-Block" für die Organisation. Weitere Informationen finden Sie unter [Konfigurieren der Verbindungsfilterung.](configure-the-connection-filter-policy.md)|
|**Spamverwaltung**||
|Antispamrichtlinien|Jede Organisation verfügt über eine Standardmäßige Antispamrichtlinie, die für alle Empfänger gilt. Für eine höhere Granularität können Sie benutzerdefinierte Antispamrichtlinien erstellen, die für bestimmte Benutzer, Gruppen oder Domänen in Ihrer Organisation gelten. Benutzerdefinierte Richtlinien werden immer vor der Standardrichtlinie angewendet, Sie können jedoch die Reihenfolge ändern, in der benutzerdefinierte Richtlinien angewendet werden. <p> Sie können die folgenden Einstellungen in Antispamrichtlinien konfigurieren: <ul><li>**Spamfilter-Verdingungsaktionen:** Wenn eine Nachricht erkannt wird, können Sie die zu ergreifende Aktion (Löschen, Verschieben in Junk-E-Mail-Ordner, Quarantäne usw.) basierend auf dem Urteil konfigurieren.</li><li>**Einstellungen für erweiterten Spamfilter (Advanced** Spam Filter, ASF): Diese Einstellungen werden unter [Advanced Spam Filter (ASF) settings in EOP beschrieben.](advanced-spam-filtering-asf-options.md)</li><li>**ZAP für Phishing und Spam:** Rückwirkende Quarantäne oder Verschieben von zugestellten Nachrichten in den Junk-E-Mail-Ordner. Weitere Informationen finden Sie unter [Zero-Hour Auto Purge (ZAP) in Exchange Online](zero-hour-auto-purge.md).</li><li>**Aktivieren von Spambenachrichtigungen für Endbenutzer**: [Erfahren Sie mehr über Spambenachrichtigungen für Endbenutzer]. (use-spam-notifications-to-release-and-report-quarantined-messages.md)</li>**Zugelassene und blockierte** Absender und Domänen: Wichtige Informationen zur sicheren Verwendung dieser Listen finden Sie unter [Create safe sender lists](create-safe-sender-lists-in-office-365.md) and Create blocked sender [lists.](create-block-sender-lists-in-office-365.md)</li><li>**Internationale Spameinstellungen:** Nachrichten basierend auf Sprache oder Quellland blockieren.</li></ul> <p> Weitere Informationen finden Sie unter [Konfigurieren von Anti-Spam-Richtlinien](configure-your-spam-filter-policies.md).|
|Verwalten von Spam Outlook oder Outlook im Web (früher bekannt als Outlook Web App)|Benutzer und Administratoren können persönliche Listen sicherer Absender und blockierte Absenderlisten in Exchange Online erstellen. Weitere Informationen finden Sie unter [Informationen zu Junk-E-Mail-Einstellungen in Outlook](configure-junk-email-settings-on-exo-mailboxes.md#about-junk-email-settings-in-outlook). <p> Wenn Sie EOP verwenden, um lokale Exchange zu schützen, sollten Sie die Verzeichnissynchronisierung verwenden, um sicherzustellen, dass diese Einstellungen mit dem Dienst synchronisiert werden. Weitere Informationen finden Sie unter [Verwalten von E-Mail-Benutzern durch Verzeichnissynchronisierung](/exchange/standalone-eop/manage-mail-users-in-eop#synchronize-directories-with-azure-active-directory-connect-aad-connect).|

|**Nachrichtenflussregeln**||
|Nachrichtenflussregeln in EOP|Praktisch alle Bedingungen, Ausnahmen und Aktionen, die in Nachrichtenflussregeln (auch als Transportregeln bezeichnet) in Exchange Online verfügbar sind, sind auch in eigenständigen EOP-Organisationen ohne Exchange Online verfügbar. Weitere Informationen zu Nachrichtenflussregeln finden Sie in den folgenden Themen: <ul><li>[Nachrichtenflussregeln](/exchange/security-and-compliance/mail-flow-rules/mail-flow-rules)</li><li>[Nachrichtenflussregelbedingungen und -ausnahmen](/exchange/security-and-compliance/mail-flow-rules/conditions-and-exceptions)</li><li>[Aktionen für Nachrichtenflussregeln](/exchange/security-and-compliance/mail-flow-rules/mail-flow-rule-actions)</li></ul>|
|Szenarien für Nachrichtenflussregel|Viele Szenarien stehen mithilfe von Transportregeln zur Verfügung. Beispiel: <ul><li>**Richtlinienbasierte Filterung und** Aktionen: Sie können nach Domäne, Schlüsselwort, Dateinamen, Dateityp, Betreffzeile, Nachrichtentext, Absender, Empfänger, Kopfzeile und IP-Adresse filtern.</li><li>**Nach Textmustern filtern:** Nachrichtenflussregeln können ein Array oder reguläre Ausdrücke verwenden, um Text zu entsprechen. Sie können auch eine Zeichenfolge oder ein Array von Zeichenfolgen verwenden, um Nachrichteneigenschaften wie beispielsweise Adresse, Betreff, Nachrichtentext oder Anlagennamen abzugleichen.</li><li>**Benutzerdefinierte Wörterbücher:** Nachrichtenflussregeln können lange Listen mit Text und Schlüsselwörtern enthalten, die die gleiche Funktionalität wie ein benutzerdefiniertes Wörterbuch bieten.</li><li>**Richtlinienregeln** pro Domäne: Sie können den Bereich einer Nachrichtenflussregel anpassen, um Absender- oder Empfängerdomänennamen, IP-Adressbereiche, Adressschlüsselwörter oder -muster, Gruppenmitgliedschaften und andere Bedingungen zu erfüllen.</li><li>**Anlagenprüfung:** Sie können Nachrichtenflussregeln erstellen, um den Dateinamen, die Erweiterung und den Inhalt von E-Mail-Anlagen zu überprüfen.</li><li>Senden von Richtlinienregelbenachrichtigungen an den **Absender:** Sie können Nachrichten ablehnen und einen Unzustellbarkeitsbericht (auch  als Unzustellbarkeitsbericht bezeichnet) über die Nachricht mit der Erklärung ablehnen oder die Nachricht mit der erweiterten Statuscodeaktion ablehnen an den Absender senden. </li><li>**Umleiten oder Kopieren von Nachrichten:** Nachrichtenflussregeln können umleiten, Empfänger von Cc oder Bcc hinzufügen, einfach Empfänger und andere Optionen hinzufügen.</li><li>**Filtern von Nachrichten und Ändern der** Nachrichtenattribute oder des Routings: Sie können Nachrichten basierend auf einer Vielzahl von Bedingungen filtern und dann eine Reihe von Aktionen auf jede Nachricht anwenden.</li><li>**Ändern der Spamsicherheitsstufe (Spam Confidence Level, SCL) von Nachrichten, die sich in der Übertragung**</li></ul> <p> Einen bestimmten Artikel zu Nachrichtenflussregelszenarios finden Sie unter [Nachrichtenflussregelprozeduren](/exchange/security-and-compliance/mail-flow-rules/mail-flow-rule-procedures).|

|**Berichterstellung und Protokollierung**||
|Nachrichtenablaufverfolgung|Administratoren können E-Mail-Nachrichten beim Passieren des Diensts verfolgen. Sie können bestimmen, ob eine gezielte E-Mail-Nachricht vom Dienst empfangen, abgelehnt, zurückgestellt oder zugestellt wurde. So können Sie in effizienter Weise Fragen der Benutzer beantworten, Probleme mit dem Nachrichtenfluss behandeln und Richtlinienänderungen überprüfen und müssen seltener den technischen Support um Unterstützung bitten. Weitere Informationen finden Sie unter [Nachrichtenablaufverfolgung im Security & Compliance Center](message-trace-scc.md).|
|Webbasierte Berichte|Die E-Mail-Schutzberichte im Security & Compliance Center stellen Messagingdaten zur Verfügung. Sie können beispielsweise überwachen, wie viel Spam und Schadsoftware erkannt wird oder wie oft Ihre Nachrichtenflussregeln übereinstimmen. Mithilfe dieser interaktiven Berichte erhalten Sie schnell einen visuellen Überblick über die zusammengefassten Daten und können Details zu einzelnen Nachrichten der letzten 90 Tage überprüfen. Weitere Informationen finden Sie unter [Verwenden von E-Mail-Schutzberichten zum Anzeigen von Daten zu Schadsoftware, Spam und Regelerkennungen.](/exchange/monitoring/use-mail-protection-reports)|
|Überwachungsprotokollierung|Weitere Informationen finden Sie unter [Überwachung von Berichten in Exchange Online](/exchange/security-and-compliance/exchange-auditing-reports/exchange-auditing-reports).|
