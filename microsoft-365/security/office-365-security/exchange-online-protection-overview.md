## <a name="how-eop-works"></a><span data-ttu-id="06455-115">Funktionsweise von EOP</span><span class="sxs-lookup"><span data-stu-id="06455-115">How EOP works</span></span>

<span data-ttu-id="06455-116">Die Funktionsweise von EOP lässt sich am besten an der Verarbeitung eingehender E-Mails veranschaulichen:</span><span class="sxs-lookup"><span data-stu-id="06455-116">To understand how EOP works, it helps to see how it processes incoming email:</span></span>

:::image type="content" source="../../media/tp_emailprocessingineopt3.png" alt-text="Grafik von E-Mails aus dem Internet oder Kundenfeedback, die an EOP und über die Verbindung, Antischadsoftware, Richtlinienfilterung und Inhaltsfilterung für den E-Mail-Fluss vor der Bewertung von Junk-E-Mails oder Quarantäne oder der Zustellung von Endbenutzer-E-Mails übergeben werden.":::

1. <span data-ttu-id="06455-118">Wenn eine eingehende Nachricht in EOP eingeht, wird zunächst die Verbindungsfilterung durchlaufen, die den Ruf des Absenders überprüft.</span><span class="sxs-lookup"><span data-stu-id="06455-118">When an incoming message enters EOP, it initially passes through connection filtering, which checks the sender's reputation.</span></span> <span data-ttu-id="06455-119">Der Großteil der Spamnachrichten wird zu diesem Zeitpunkt beendet und von EOP abgelehnt.</span><span class="sxs-lookup"><span data-stu-id="06455-119">The majority of spam is stopped at this point and rejected by EOP.</span></span> <span data-ttu-id="06455-120">Weitere Informationen finden Sie unter [Konfigurieren der Richtlinie für Verbindungsfilter](configure-the-connection-filter-policy.md).</span><span class="sxs-lookup"><span data-stu-id="06455-120">For more information, see [Configure connection filtering](configure-the-connection-filter-policy.md).</span></span>

2. <span data-ttu-id="06455-121">Anschließend wird die Nachricht auf Schadsoftware überprüft.</span><span class="sxs-lookup"><span data-stu-id="06455-121">Then the message is inspected for malware.</span></span> <span data-ttu-id="06455-122">Wenn in der Nachricht oder den Anlagen Schadsoftware gefunden wird, wird die Nachricht nur an einen Administrator-Quarantänespeicher weitergeleitet.</span><span class="sxs-lookup"><span data-stu-id="06455-122">If malware is found in the message or the attachment(s) the message is routed to an admin only quarantine store.</span></span> <span data-ttu-id="06455-123">Weitere Informationen zum Schutz vor Schadsoftware finden Sie unter [Antischadsoftwareschutz in EOP.](anti-malware-protection.md)</span><span class="sxs-lookup"><span data-stu-id="06455-123">To learn more about malware protection, see [Anti-malware protection in EOP](anti-malware-protection.md).</span></span>

3. <span data-ttu-id="06455-124">Die Nachricht wird durch Richtlinienfilterung fortgesetzt, wobei sie anhand aller von Ihnen erstellten Nachrichtenflussregeln (auch als Transportregeln bezeichnet) ausgewertet wird.</span><span class="sxs-lookup"><span data-stu-id="06455-124">The message continues through policy filtering, where it's evaluated against any mail flow rules (also known as transport rules) that you've created.</span></span> <span data-ttu-id="06455-125">Beispielsweise kann eine Regel eine Benachrichtigung an einen Vorgesetzten senden, wenn eine Nachricht von einem bestimmten Absender eingeht.</span><span class="sxs-lookup"><span data-stu-id="06455-125">For example, a rule can send a notification to a manager when a message arrives from a specific sender.</span></span>

   <span data-ttu-id="06455-126">In der lokalen Organisation mit Exchange Enterprise CAL mit Dienstlizenzen erfolgen zu diesem Zeitpunkt auch [DLP-Prüfungen (Data Loss Prevention, Verhinderung von Datenverlust)](/exchange/security-and-compliance/data-loss-prevention/data-loss-prevention) in EOP.</span><span class="sxs-lookup"><span data-stu-id="06455-126">In on-premises organization with Exchange Enterprise CAL with Services licenses, [Data loss prevention (DLP)](/exchange/security-and-compliance/data-loss-prevention/data-loss-prevention) checks in EOP also happen at this point.</span></span>

4. <span data-ttu-id="06455-127">Die Nachricht wird durch die Inhaltsfilterung (Antispam und Antispoofing) übergeben, bei der schädliche Nachrichten als Spam, Spam mit hoher Spamwahrscheinlichkeit, Phishing, Phishing mit hoher Vertrauenswürdigkeit oder massenweise (Antispamrichtlinien) oder Spoofing (Spoofingeinstellungen in Antiphishingrichtlinien) identifiziert werden.</span><span class="sxs-lookup"><span data-stu-id="06455-127">The message passes through content filtering (anti-spam and anti-spoofing) where harmful messages are identified as spam, high confidence spam, phishing, high confidence phishing, or bulk (anti-spam policies) or spoofing (spoof settings in anti-phishing policies).</span></span> <span data-ttu-id="06455-128">Sie können die für die Nachricht auszuführende Aktion basierend auf dem Filterbewertungsausdruck (Quarantäne, Verschieben in den Junk-E-Mail-Ordner usw.) konfigurieren.</span><span class="sxs-lookup"><span data-stu-id="06455-128">You can configure the action to take on the message based on the filtering verdict (quarantine, move to the Junk Email folder, etc.).</span></span> <span data-ttu-id="06455-129">Weitere Informationen finden Sie unter [Konfigurieren von Antispamrichtlinien](configure-your-spam-filter-policies.md) und [Konfigurieren von Antiphishingrichtlinien in EOP.](configure-anti-phishing-policies-eop.md)</span><span class="sxs-lookup"><span data-stu-id="06455-129">For more information, see [Configure anti-spam policies](configure-your-spam-filter-policies.md) and [Configure anti-phishing policies in EOP](configure-anti-phishing-policies-eop.md).</span></span>

<span data-ttu-id="06455-130">Eine Nachricht, die alle diese Schutzebenen erfolgreich übergibt, wird an die Empfänger übermittelt.</span><span class="sxs-lookup"><span data-stu-id="06455-130">A message that successfully passes all of these protection layers is delivered to the recipients.</span></span>

<span data-ttu-id="06455-131">Weitere Informationen finden Sie unter [Reihenfolge und Rangfolge des E-Mail-Schutzes.](how-policies-and-protections-are-combined.md)</span><span class="sxs-lookup"><span data-stu-id="06455-131">For more information, see [Order and precedence of email protection](how-policies-and-protections-are-combined.md).</span></span>
