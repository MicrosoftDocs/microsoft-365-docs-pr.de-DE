2. <span data-ttu-id="a1bdc-121">Anschließend wird die Nachricht auf Schadsoftware überprüft.</span><span class="sxs-lookup"><span data-stu-id="a1bdc-121">Then the message is inspected for malware.</span></span> <span data-ttu-id="a1bdc-122">Wenn schadsoftware in der Nachricht oder in den Anlagen gefunden wird, wird die Nachricht an einen Nur-Administrator-Quarantänespeicher geroutet.</span><span class="sxs-lookup"><span data-stu-id="a1bdc-122">If malware is found in the message or the attachment(s) the message is routed to an admin only quarantine store.</span></span> <span data-ttu-id="a1bdc-123">Weitere Informationen zum Schutz vor Schadsoftware finden Sie unter [An malware protection in EOP](anti-malware-protection.md).</span><span class="sxs-lookup"><span data-stu-id="a1bdc-123">To learn more about malware protection, see [Anti-malware protection in EOP](anti-malware-protection.md).</span></span>

3. <span data-ttu-id="a1bdc-124">Die Nachricht wird über die Richtlinienfilterung fortgesetzt, wobei sie anhand der von Ihnen erstellten Nachrichtenflussregeln (auch als Transportregeln bezeichnet) ausgewertet wird.</span><span class="sxs-lookup"><span data-stu-id="a1bdc-124">The message continues through policy filtering, where it's evaluated against any mail flow rules (also known as transport rules) that you've created.</span></span> <span data-ttu-id="a1bdc-125">Beispielsweise kann eine Regel eine Benachrichtigung an einen Vorgesetzten senden, wenn eine Nachricht von einem bestimmten Absender eintrifft.</span><span class="sxs-lookup"><span data-stu-id="a1bdc-125">For example, a rule can send a notification to a manager when a message arrives from a specific sender.</span></span>

   <span data-ttu-id="a1bdc-126">In einer lokalen Organisation mit Exchange Enterprise CAL mit Services-Lizenzen finden an diesem Punkt auch [DLP-Prüfungen (Data Loss Prevention,](/exchange/security-and-compliance/data-loss-prevention/data-loss-prevention) Verhinderung von Datenverlust) in EOP statt.</span><span class="sxs-lookup"><span data-stu-id="a1bdc-126">In on-premises organization with Exchange Enterprise CAL with Services licenses, [Data loss prevention (DLP)](/exchange/security-and-compliance/data-loss-prevention/data-loss-prevention) checks in EOP also happen at this point.</span></span>

4. <span data-ttu-id="a1bdc-127">Die Nachricht durchläuft die Inhaltsfilterung (Antispam und Antis spoofing), bei der schädliche Nachrichten als Spam, Spam mit hoher Vertrauenssicherheit, Phishing, Phishing mit hoher Vertrauenssicherheit oder Massen (Antispamrichtlinien) oder Spoofing (Spoofeinstellungen in Antiphishingrichtlinien) identifiziert werden.</span><span class="sxs-lookup"><span data-stu-id="a1bdc-127">The message passes through content filtering (anti-spam and anti-spoofing) where harmful messages are identified as spam, high confidence spam, phishing, high confidence phishing, or bulk (anti-spam policies) or spoofing (spoof settings in anti-phishing policies).</span></span> <span data-ttu-id="a1bdc-128">Sie können die Aktion konfigurieren, die für die Nachricht basierend auf dem Filterverkündungs-Urteil (Quarantäne, Verschieben in den Junk-E-Mail-Ordner usw.) zu ergreifen ist.</span><span class="sxs-lookup"><span data-stu-id="a1bdc-128">You can configure the action to take on the message based on the filtering verdict (quarantine, move to the Junk Email folder, etc.).</span></span> <span data-ttu-id="a1bdc-129">Weitere Informationen finden Sie unter [Konfigurieren von Antispamrichtlinien und](configure-your-spam-filter-policies.md) Konfigurieren von [Antiphishingrichtlinien in EOP](configure-anti-phishing-policies-eop.md).</span><span class="sxs-lookup"><span data-stu-id="a1bdc-129">For more information, see [Configure anti-spam policies](configure-your-spam-filter-policies.md) and [Configure anti-phishing policies in EOP](configure-anti-phishing-policies-eop.md).</span></span>
