# <a name="investigate-and-respond-using-microsoft-365-defender-in-a-pilot-environment"></a>Untersuchen und Reagieren mithilfe von Microsoft 365 Defender in einer Pilotumgebung

**Gilt für:**
- Microsoft 365 Defender

In diesem Artikel wird beschrieben, wie Sie Vorfälle mit Angriffssimulationen und Lernprogrammen erstellen und Microsoft 365 Defender verwenden, um zu untersuchen und zu reagieren. Bevor Sie mit diesem Prozess beginnen, stellen Sie sicher, dass Sie den allgemeinen Prozess für die [Auswertung von Microsoft 365 Defender](eval-overview.md) überprüft und die Microsoft 365 Defender [Evaluierungsumgebung erstellt](eval-create-eval-environment.md)haben.

Führen Sie die folgenden Schritte aus.

![Schritte zum Ausführen simulierter Vorfallreaktionen in der Microsoft 365 Defender Evaluierungsumgebung](../../media/eval-defender-investigate-respond/eval-defender-eval-investigate-respond-steps.png)

In der folgenden Tabelle werden die Schritte in der Abbildung beschrieben.

| |Schritt  |Beschreibung  |
|---------|---------|---------|
|1|[Angriffe simulieren](eval-defender-investigate-respond-simulate-attack.md)     |   Simulieren Sie Angriffe auf Ihre Evaluierungsumgebung, und verwenden Sie das Microsoft 365 Defender-Portal, um auf Vorfälle zu reagieren.      |
|2|[Testen der Funktionen für die Reaktion auf Vorfälle ](eval-defender-investigate-respond-additional.md)    |    Testen Sie Features und Funktionen in Microsoft 365 Defender.     |
