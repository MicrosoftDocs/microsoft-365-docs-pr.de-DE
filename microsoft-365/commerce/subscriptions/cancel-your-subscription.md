# <a name="cancel-your-subscription"></a>Kündigen Ihres Abonnements

*Berechtigung:* Wenn Benutzern weniger als 25 Lizenzen zugewiesen sind, können Sie Ihre Office 365 für Unternehmen Test-oder kostenpflichtigen Abonnements online im Admin Center jederzeit kündigen. Wenn Sie über mehr als 25 Lizenzen für Benutzer verfügen, [rufen Sie den Support an, um Ihr Abonnement zu kündigen](../../admin/contact-support-for-business-products.md).

*Rückerstattung:* Jeder anteilige Kredit wird Ihnen innerhalb des nächsten Abrechnungszyklus zurückgegeben.

> [!NOTE]
> Wenn Sie mehrere Abonnements für dasselbe Produkt wie Office 365 Enterprise E3 haben, wirkt sich das Abbrechen eines dieser Produkte nicht auf die erworbenen Lizenzen oder Dienste innerhalb der anderen Abonnements aus.

## <a name="steps-to-cancel-your-subscription"></a>Schritte zum Kündigen des Abonnements

Wenn Sie Ihren eigenen Domänennamen zur Verwendung mit Ihrem Abonnement hinzugefügt haben, müssen Sie die Domäne entfernen, bevor Sie Ihr Abonnement kündigen. Weitere Informationen finden Sie unter [Entfernen einer Domäne aus Office 365](../../admin/get-help-with-domains/remove-a-domain.md).

::: moniker range="o365-worldwide"

> [!NOTE]
> Wenn Sie das neue Microsoft 365 Admin Center nicht verwenden, können Sie es aktivieren, indem Sie den Umschalter **Das neue Admin Center** am oberen Rand der Startseite auswählen.

1. Navigieren Sie im Admin Center zur Seite **Abrechnung** \> <a href="https://go.microsoft.com/fwlink/p/?linkid=842054" target="_blank">Produkte und Dienste</a>.

2. Suchen Sie das Abonnement, das Sie abbrechen möchten, und wählen Sie unter **Einstellungen & Aktionen**die Option **Abonnement kündigen**aus.

3. Überprüfen Sie die wichtigen Termine, geben Sie Feedback dazu, warum Sie abbrechen möchten, und wählen Sie dann **Abonnement kündigen**aus.

## <a name="other-steps-you-might-have-to-take"></a>Andere Schritte, die Sie möglicherweise ausführen müssen

### <a name="change-custom-domain-settings"></a>Ändern von benutzerdefinierten Domäneneinstellungen

Wenn Sie eine benutzerdefinierte Domäne mit Ihrem Abonnement verwenden, müssen Sie einige zusätzliche Schritte ausführen, bevor Sie Ihr Abonnement kündigen können. Wenn Sie keine benutzerdefinierte Domäne haben, können Sie mit dem [Speichern Ihrer Daten](#save-your-data)fortfahren.

#### <a name="change-your-domain-nameserver-records-if-needed"></a>Ändern ihrer Domänennamenserver-Einträge (falls erforderlich)

Wenn Sie eine benutzerdefinierte Domäne eingerichtet haben, haben Sie DNS-Einträge hinzugefügt, damit die Domäne mit Office 365 Diensten funktionieren würde. Bevor Sie Ihre Domäne entfernen, müssen Sie die DNS-Einträge wie Ihren Domänen-MX-Eintrag auf Ihrem DNS-Host aktualisieren.

Ändern Sie beispielsweise den MX-Eintrag auf Ihrem DNS-Host. E-Mail-Nachrichten, die an Ihre Domäne gesendet werden, werden nicht an Ihre Office 365 Adresse weitergeleitet und stattdessen an Ihren neuen e-Mail-Anbieter (Mit einem MX-Eintrag wird festgelegt, wohin E-Mails für Ihre Domäne gesendet werden.)

- Wenn Ihre Namenservereinträge (NS) [auf Office 365 Namenserver zeigen, werden](../../admin/setup/add-domain.md)Änderungen am MX-Eintrag erst wirksam, wenn Sie Ihre NS-Einträge so ändern, dass Sie auf Ihren neuen DNS-Host zeigen (siehe Schritt 2).

- Informieren Sie vor dem Aktualisieren des MX-Eintrags Ihre Benutzer über das Datum, an dem Sie Ihre e-Mails umschalten möchten, und den neuen e-Mail-Anbieter, den Sie verwenden möchten. Wenn Ihre Benutzer ihre vorhandenen Office 365-e-Mails an den neuen Anbieter weiterleiten möchten, müssen Sie außerdem zusätzliche Schritte ausführen.

- Folgen Sie an dem Tag, an dem Sie den MX-Eintrag ändern, den restlichen Schritten in diesem Artikel.

#### <a name="update-your-domain-mx-and-other-dns-records-if-youre-using-a-custom-domain"></a>Aktualisieren ihrer Domänen-MX-und anderer DNS-Einträge (wenn Sie eine benutzerdefinierte Domäne verwenden)

Wenn Sie Ihre Namenservereinträge (NS) zu Office 365 beim Einrichten Ihrer Domäne gewechselt haben, müssen Sie den MX-Eintrag und andere DNS-Einträge auf dem zu verwendenden DNS-Host einrichten oder aktualisieren und dann den NS-Eintrag auf diesen DNS-Host ändern.

Wenn Sie bei der Einrichtung Ihrer Domäne keine NS-Einträge gewechselt haben, wird beim Ändern des MX-Eintrags Ihre e-Mail sofort an die neue Adresse weitergegeben.

Weitere Informationen finden Sie unter [wie kann Office 365 meine DNS-Einträge verwalten?](../../admin/setup/domains-faq.md#how-does-office-365-manage-my-dns-records). Informationen zum Ändern der NS-Datensätze finden Sie unter [Entfernen einer Domäne aus Office 365](../../admin/get-help-with-domains/remove-a-domain.md).

### <a name="save-your-data"></a>Speichern Ihrer Daten

Wenn die Löschung wirksam wird, verlieren Ihre Benutzer den Zugriff auf Ihre Daten. Bevor Sie das Abonnement kündigen, müssen Sie die OneDrive für Unternehmen-oder SharePoint Online-Dateien an einem anderen Speicherort speichern. Alle Kundendaten, die Sie zurücklassen, werden möglicherweise nach 30 Tagen gelöscht und spätestens 180 Tage nach dem Abbruch gelöscht.

- Informationen zum Verschieben von E-Mails, Kontakten, Aufgaben und Kalenderinformationen in ein anderes Konto finden Sie unter [Exportieren oder Sichern von E-Mails, Kontakten und Kalendern in einer Outlook PST-Datei](https://support.office.com/article/14252b52-3075-4e9b-be4e-ff9ef1068f91.aspx).

- Informationen zum Speichern einer Dokumentbibliothek oder von Listeninhalten (z. B. Kontakten) aus einer SharePoint Online-Umgebung (OneDrive for Business oder Teamwebsites) auf Dateifreigaben oder einem lokalen Computer finden Sie unter [Manuelle Migration von SharePoint Online-Inhalten](https://support.microsoft.com/kb/2783484).

### <a name="uninstall-office-optional"></a>Deinstallieren von Office (optional)

Wenn Sie Ihr Abonnement storniert haben und die Benutzer nicht in ein anderes Abonnement mit Office umgezogen haben, wird Office 365 im Modus mit eingeschränkter Funktionalität ausgeführt. In diesem Fall können Benutzer nur Dokumente lesen und drucken, und in Office 365 Anwendungen werden nicht [lizenzierte Produktbenachrichtigungen](https://support.office.com/article/0d23d3c0-c19c-4b2f-9845-5344fedc4380.aspx)angezeigt. Um Verwirrung zu vermeiden, weisen Sie die Benutzer an, [Office von ihren Computern zu deinstallieren](https://support.office.com/article/9dd49b83-264a-477a-8fcc-2fdf5dbf61d8.aspx).
