# <a name="microsoft-managed-desktop-roles-and-responsibilities"></a>Microsoft verwalteter Desktop Rollen und Aufgaben


<!--This topic is the target for a "Learn more" link in the Admin Portal (aka.ms/admin-access); do not delete.-->
<!-- from Roles and responsibilities -->

Wenn Ihre Organisation Microsoft verwalteter Desktop registriert ist, tun was Microsoft für Sie? Und was sind Ihrer Organisation Zuständigkeiten?

## <a name="microsofts-roles-and-responsibilities"></a>Microsoft Rollen und Aufgaben

Im folgenden sind einige wichtige Rollen und Aufgaben, die Ihnen von Microsoft bereitgestellt werden.

Rolle oder Verantwortung | Beschreibung
--- | ---
MDM Gruppenrichtlinien-Verwaltungskonsole | Microsoft wird Anwenden von MDM Richtlinien nach den bewährten Methoden und Anforderungen für die Richtlinie Änderungen berücksichtigen. Es wird auch Ihres Mandanten wie in [Geräterichtlinien](../service-description/device-policies.md)vorgeschrieben ändern.
App-Bereitstellung | Microsoft stellt genehmigte Anwendungen an Intune mithilfe des Clients Microsoft verwalteter Desktop-Geräte bereit. Weitere Informationen finden Sie unter [Apps](../get-ready/apps.md). 
Support für Endbenutzer | Microsoft bietet den Endbenutzersupport für Geräte, Windows und Office Produktsuite für alle registrierten Benutzer. 
Unterstützung für den Microsoft verwalteten Desktops | Microsoft bietet Unterstützung für Kunden IT Abteilungen über ein Desktop Vorgänge Team von Microsoft verwaltet. Dieses Team wird unterstützen die Problembehandlung bei technischen, Anfragen und Incident Management für die Umgebung des Kunden Microsoft verwalteter Desktop zu ändern. Weitere Informationen finden Sie unter [Unterstützung erhalten möchten](../service-description/support.md).
Überwachen der Sicherheit | Microsoft überwacht Customer Microsoft verwalteter Desktop Geräte mit Windows Defender ATP. Wenn eine Bedrohung erkannt wird, durch die Microsoft verwaltete Desktops Security Operations Center (SOC), Microsoft benachrichtigt der "Kunde", das Gerät zu isolieren und Remote, das Problem zu beheben. Weitere Informationen finden Sie unter [Sicherheit](../service-description/security.md).
Aktualisieren der Überwachung und Verwaltung | Microsoft überwacht aktiv Kunden Microsoft verwalteter Desktop Geräte, um sicherzustellen, dass die neuesten Updates Qualität, Features und Definition für Microsoft Windows und Microsoft Office installiert sind. Weitere Informationen finden Sie unter [wie Updates behandelt werden](../service-description/updates.md).
Gerät Beschaffung | Microsoft wird Drop-geordnete verwalteter Microsoft-Desktop-Geräte an die Endbenutzer und IT-Verteilungspunkt Ihrer Wahl liefern. Weitere Informationen finden Sie unter [Geräte](../get-started/devices.md).

## <a name="your-roles-and-responsibilities"></a>Rollen und Aufgaben

Es folgt eine zusätzliche Reihe von allgemeinen Rollen, die nicht von Microsoft bereitgestellt werden, aber für eine erfolgreiche Bereitstellung erforderlich sind. Es ist nicht vollständig aber wird in der Regel für die meisten Organisationen. 

Rolle oder reponsibility | Beschreibung
--- | ---
Änderungsmanagement | Kunden, werden von Microsoft verwalteten Desktops im Voraus benachrichtigt werden, wenn Änderungen für ihre Umgebung Microsoft verwalteter Desktop getroffen werden müssen. Kunden benötigen, ihre eigenen Änderungsmanagementprozesses und sind erforderlich, um einen Kontakt mit Microsoft verwalteten Desktops haben. Der Kunde ist erforderlich, um Ressourcen überprüfen und genehmigen diese Änderungen haben. Weitere Informationen finden Sie unter [Operations und überwachen](../service-description/operations-and-monitoring.md).  
Identitätsverwaltung | Erstellen von Benutzerkonten, Zuweisung von Benutzern zu Gruppen und Metadaten auf dem aktuellen Stand zu halten. 
Office 365-Konfiguration und Verwaltung | Verwaltung von Exchange-Oonline, einschließlich:<br>-E-Mail Verwaltung<br>-Mailbox und Regel Konfiguration<br>-Lokale Exchange-Verwaltung<br><br>Tools für die Zusammenarbeit, für die SharePoint Server, Domänenmanagement, Sicherheit und Informationen Richtlinien, die in Office 365 Admin-Portal (Microsoft verwalteten Desktops wird sichergestellt, Office-Anwendungen in Endbenutzer Geräte bereitgestellt werden und auf dem aktuellen Stand bleiben) festgelegt. 
Support für Endbenutzer | Der Kunde bietet Unterstützung für Endbenutzer: <br>-Klicken Sie auf Website-Infrastruktur: alle Netzwerk und Internet Connectivity, VPN-Infrastruktur und Clientkonfiguration, lokalen Konferenz Raum Equipment, Drucker, Proxyserver und Firewalls-Konfiguration.<br><br>-Unternehmensweite Cloudressourcen: e-Mail, SharePoint, Collaboration-Dienste und andere Cloud-Infrastruktur, die auf der unternehmensweiten Technologie Speicherbedarf bezieht.<br><br>-Line-of-Business Applications: benutzerdefinierte Software, 3rd – Software von Drittanbietern, Enterprise Resource planning (ERP)-Software, Entwurfstools und nichts in diesem Dokument nicht angegeben.
Benutzer und Gerät gruppieren | Microsoft verwalteter Desktop Betriebsteams erstellen und als Teil der IT-Vorgänge erforderlich Gerät und Benutzergruppen verwalten. Der Kunde wird nicht ohne ersten Kontaktaufnahme mit dem IT-Vorgänge über unterstützte Kanäle dieser Gruppen ändern. Alle erkannt, dass die Änderungen rückgängig gemacht werden soll.
Apps | Kunden werden eine Liste der apps bereitzustellen, die sie in ihrer Organisation (neben apps, die mit Microsoft-365-Lizenz enthalten) verwenden möchten. Kunden bieten ein bereitstellbares Paket (Appx oder MSI-Datei)<br>Der Kunde ist verantwortlich für:<br>-App Lizenzmanagement – mit den richtigen Typ und die Anzahl der Lizenzen<br>-App Zuweisung – Zuweisen von apps Azure AD-Benutzergruppen<br>-App Updates – Überwachung, zum Packen und Bereitstellen von Feature und app-Updates<br>-Benutzer Anwendungstests (Benutzerakzeptanztest)<br><br>Weitere Informationen finden Sie unter [Apps](../get-ready/apps.md)
Überwachung der Sicherheit und die Antwortzeit | Wenn sicherheitsrelevanten Ereignisses erkannt wird, die außerhalb des Bereichs des Microsoft verwalteten Desktops Operations ist benötigen wir eine Liste der bevorzugten Kundenkontakte basierend auf den Schweregrad des Problems.<br><br>Der Kunde ist verantwortlich für die Untersuchung und Beheben von Vorfälle für nicht - Microsoft verwalteter Desktop-Geräte und sicherstellen, dass das Microsoft verwalteten Desktops Vorgänge Team mögliche Fehler informiert wird, die den Dienst auswirken können.
Unterstützung von Vorgängen | Microsoft verwalteten Desktops Operations benötigt eine Liste der bevorzugten Kundenkontakte und Fachleuten. Wir brauchen diese für den Fall, dass ein nicht - Microsoft verwalteter Desktop betriebliche Vorfall vorhanden ist. <br><br>Der Kunde ist verantwortlich für die Untersuchung und Beheben von Vorfälle für nicht - Microsoft verwalteter Desktop-Geräte und Dienste und sicherstellen, dass Microsoft verwalteten Desktops Vorgänge Team immer darüber informiert ist.
Netzwerkinfrastruktur, einschließlich VPN | Installation, Konfiguration und Verwaltung (einschließlich Problembehandlung und Debuggen) alle netzwerkbezogene Infrastruktur und Dienste, einschließlich Internet Connectivity Netzwerk-Steuerelemente, Proxykonfiguration und remote Connectivity Infrastruktur.<br><br>In der Standardeinstellung Microsoft verwalteter Desktop nicht angeben oder einen Proxy erfordern. Wenn eine (in Hardware- oder) konfiguriert ist, besteht jedoch eine Auflistung von URLs, die von den Proxy zugelassen werden muss. Der Kunde ist verantwortlich für die Problembehandlung bei Konflikte oder Inkompatibilitäten aufgrund mehrere Proxys.<br><br>Weitere Informationen finden Sie unter [Proxy-Konfiguration](../get-ready/network.md).
Drucken | Installieren, verwalten und Verwalten von Druckern und print Warteschlangen. Cloud Drucken wird empfohlen, ist jedoch nicht erforderlich. 
Mobile Geräte | Geräte und Zubehör, die nicht von Microsoft verwaltet Desktop bereitgestellt wird. Verwaltete Desktops Operations-Team von Microsoft unterstützt nur Microsoft verwalteter Desktop Gerät, Dockingstation und Zubehör enthalten.
