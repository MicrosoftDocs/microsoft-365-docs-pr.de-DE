# <a name="microsoft-365-lighthouse-tenant-list-overview"></a>Übersicht über Microsoft 365 Lighthouse Mandantenliste

> [!NOTE]
> Die in diesem Artikel beschriebenen Features befinden sich in der Vorschau, können geändert werden und sind nur für Partner verfügbar, die die [Anforderungen](m365-lighthouse-requirements.md)erfüllen. Wenn Ihre Organisation nicht über Microsoft 365 Lighthouse verfügt, lesen Sie ["Registrieren für Microsoft 365 Lighthouse".](m365-lighthouse-sign-up.md)

Die Microsoft 365 Lighthouse Mandantenliste bietet Einblicke in die verschiedenen Mandanten, mit denen Sie einen Vertrag haben, einschließlich des Mandanten-Onboardingstatus im Verhältnis zu Microsoft 365 Lighthouse. In der Mandantenliste können Sie auch Mandanten markieren, um unterschiedliche Filter in Microsoft 365 Lighthouse bereitzustellen, und einen Drilldown ausführen, um mehr über einen bestimmten Mandanten und den Status des Bereitstellungsplans zu erfahren.

Nachdem Ihre Mandanten die [Microsoft 365 Lighthouse Onboardinganforderungen](m365-lighthouse-requirements.md)erfüllt haben, wird ihr Status in der Mandantenliste als **aktiv** angezeigt.

Um auf die Mandantenliste in Microsoft 365 Lighthouse zuzugreifen, wählen Sie **"Mandanten"** im linken Navigationsbereich aus, um die Seite "Mandanten" zu öffnen.

## <a name="tenant-status"></a>Mandantenstatus

Die folgende Tabelle zeigt die verschiedenen Statusmeldungen und deren Bedeutung.<br><br>

| Statusmeldung | Beschreibung |
|--|--|
| Aktiv | Onboarding und Datenfluss wurden gestartet. |
| In Bearbeitung | Mandant ermittelt, aber nicht vollständig integriert. |
| Nicht zulässig, DAP | Das Einrichten delegierter Administratorrechte (Delegated Admin Privileges, DAP) ist erforderlich. |
| Nicht zulässig, Benutzeranzahl | Der Mandant hat mehr Benutzer als zulässig. |
| Nicht zulässig, Lizenz | Der Mandant verfügt nicht über eine erforderliche Lizenz. |
| Inaktiv | Der Mandant ist nicht mehr aktiv. |

Nachdem Sie einen Mandanten aktiviert haben, können Sie keine Maßnahmen für den Mandanten ergreifen, während Microsoft 365 Lighthouse den Inaktivierungsprozess abgeschlossen hat. Es kann bis zu 48 Stunden dauern, bis die Deaktivierung abgeschlossen ist.

Wenn Sie einen Mandanten reaktivieren möchten, kann es bis zu 48 Stunden dauern, bis Daten erneut angezeigt werden.

## <a name="tenant-tags"></a>Mandantentags

Sie können Ihre Kundenmandanten mit einer benutzerdefinierten Bezeichnung innerhalb Microsoft 365 Lighthouse kennzeichnen. Diese Tags können verwendet werden, um Ihre Mandanten zu organisieren, und können Ihnen auch helfen, die vorhandenen Ansichten und Einblicke zu filtern, die für relevante Gruppen von Kundenmandanten verfügbar sind. Sie können Ihre Tags auch verwalten und welchen Mandanten sie auf der Seite "Mandanten" zugewiesen sind.
